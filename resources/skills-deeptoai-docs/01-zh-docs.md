# Claude Skills Explorer

一个全面探索 Claude Skills 架构、模式和实际应用的文档平台。了解 skills 如何扩展 Claude 的功能并启用自动化工作流。

__Claude Skills Explorer__ 是一个文档平台，旨在揭示 Claude Skills 的架构和实际应用 - 可执行的工作流，可扩展 Claude 的功能并实现复杂的自动化模式。

本项目同时服务于 __开发者__（构建 skills）和 __skill 使用者__（希望有效利用 skills 的人）。我们的目标是创建一个全面的知识库，连接理论与实践。

### 挑战：Skill 发现和理解

随着 Claude Skills 从实验性工具发展为生产级系统，我们发现了关键差距：虽然 skills 变得更强大和复杂，但如何有效使用它们仍然很困难。开发者苦于：

- __文档不足__：许多 skills 缺乏解释其架构的综合指南
- __发现问题__：难以找到适合特定任务的相关 skills
- __学习曲线__：理解 skill 模式和最佳实践需要大量试错
- __验证缺失__：没有标准化的方法来验证 skill 质量或有效性

### 解决方案：知识驱动的生态系统

我们创建 Claude Skills Explorer 来解决这些挑战：

1. __深入的 Skill 分析__：系统地分析每个 skill，记录其：

   - 架构和设计模式
   - 实现策略
   - 实际应用案例
   - 最佳实践和常见陷阱
2. __多语言文档__：提供英文、中文和法文内容，让全球用户都能访问 skills
3. __自动化框架__：我们构建了编写关于 skils 的技能的 skills，创建了一个自文档化的生态系统
4. __验证流程__：自动化的 MDX 验证和语义化发布确保内容质量

该文档平台本身由我们开发的 Claude Skills 提供支持。它是 skill 自动化实际应用的展示。

### 生态系统的运作

我们的项目使用三个相互关联的 skills，共同创建完全自动化的文档工作流：

#### 1. __skill-article-writer__：内容生成器

这个 meta-skill 分析现有的 Claude skills 并生成全面的教程文章。它不仅仅是文档 - 它是智能内容创建：

- __分析 skill 结构__以理解架构和模式
- __提取见解__从代码、文档和示例中
- __生成结构化文章__使用正确的 MDX 格式
- __自动创建多语言版本__

__示例工作流__：

```
# Claude 使用 skill-article-writer 理解 mcp-builder
→ 分析 SKILL.md、scripts 和 examples
→ 生成 7,300+ 字的综合指南
→ 创建英文、中文和法文版本
→ 自动更新导航
```

#### 2. __skill-article-publisher__：质量守门员

在任何文章上线之前，此 skill 通过以下方式确保质量：

- __MDX 语法验证__：检测并修复常见问题，如未转义的比较运算符
- __构建验证__：运行 `npm run build` 捕获编译错误
- __语义提交生成__：基于变更类型创建有意义的提交消息
- __自动发布__：暂存、提交并推送到远程仓库

__关键创新__：验证捕获了会破坏构建的微妙 MDX 问题，例如：

- `>` 转换为 `&gt;`（防止 XML 解析错误）
- `<` 转换为 `&lt;`
- 正确的前置结构
- 组件标签平衡

#### 3. __综合 Skill 分析__：知识库

我们已创建来自 Anthropic 仓库的生产级 skills 的详细分析：

这些不是肤浅的概述 - 它们是平均 7,000+ 字的深度技术深入探讨，包含架构分析、代码示例和实际应用。

- __analyzing-mcp-builder__：MCP 服务器开发完整指南，涵盖 4 阶段工作流、Python/TypeScript 模式和评估框架
- __analyzing-webapp-testing__：综合 Playwright 测试指南，包含决策树、侦察模式和多服务器管理

无论您是构建 skills 的开发者还是利用它们的 skill 用户，本平台都有适合您的资源。

### 核心技术

- __Next.js__：用于服务端渲染和静态生成的 React 框架
- __Fumadocs__：支持 MDX 的文档框架，具有精美的组件
- __MDX__：Markdown + JSX，用于丰富的交互式文档
- __TypeScript__：类型安全的开发
- __Tailwind CSS__：实用优先的样式

### 自动化层

5. 更改推送到仓库
