# Superpowers 研究笔记

> 研究日期：2025-12-27
> 项目地址：https://github.com/obra/superpowers

## 一句话总结

Superpowers 是一个为 Claude Code 设计的**综合性技能库**，包含 20+ 个经过实战验证的开发工作流程，将 AI 从简单助手转变为自主开发伙伴。

## 核心价值

1. **系统化开发流程**：不是工具集合，而是完整的开发理念
2. **质量优先**：强制 TDD、代码审查、调试工作流程
3. **自主能力**：子代理驱动开发，可自主工作数小时
4. **团队协作**：Git worktree、代码审查、并行开发支持

## 核心工作流程

```
brainstorming（头脑风暴）
    ↓
writing-plans（编写计划）
    ↓
using-git-worktrees（创建隔离环境）
    ↓
executing-plans（执行计划）
    ↓
test-driven-development（TDD 实现）
    ↓
requesting-code-review（代码审查）
    ↓
finishing-a-development-branch（完成分支）
```

## 关键发现

### 优点

1. **强制性流程**：技能自动触发，不是建议，确保最佳实践执行
2. **TDD 严格**：如果在测试之前写代码，会删除这些代码
3. **两阶段审查**：规范合规性 + 代码质量双重保证
4. **Git worktree 支持**：安全的并行开发，避免分支冲突
5. **自文档化**：每个决策和权衡都被记录

### 缺点

1. **学习曲线**：20+ 技能需要时间学习和适应
2. **刚性要求**：不能跳过步骤，对简单任务可能过度
3. **平台限制**：仅限 Claude Code 2.0.13+
4. **资源消耗**：子代理驱动开发消耗更多 token 和时间
5. **团队采用挑战**：需要全团队同意并改变工作流程

## 适用场景

✅ **强烈推荐**：
- 中大型项目
- 高质量要求（生产环境关键系统）
- 团队协作开发
- 长期维护项目

⚠️ **谨慎使用**：
- 小型个人项目
- 快速原型开发
- 遗留代码维护

❌ **不推荐**：
- 时间极其紧迫的项目
- 不愿意改变工作流程的团队
- 需要高度灵活性的场景

## 个人评价

**总体评分**：⭐⭐⭐⭐⭐ (4.5/5)

**为什么值得使用**：

- 不是所有场景都适用，需要判断
